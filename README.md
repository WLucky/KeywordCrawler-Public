# 多平台关键词数据采集

一个用于自动化采集多个平台数据的工具，支持关键词搜索、视频/笔记获取、评论采集及视频下载功能。当前已支持抖音、哔哩哔哩和Tavily聚合引擎，更多平台正在开发中。

## 功能特性

- 支持多个主流平台的数据采集
- 自动化搜索关键词并获取相关内容
- 数据导出为 Excel 格式
- 支持 GitHub Actions 自动化执行
- 失败自动重试机制

## 快速开始

### 环境要求

- Python 3.11+
- Node.js 20.11.0+
- Playwright 浏览器驱动

### 安装依赖

```bash
pip install -r requirements.txt
playwright install
```

### 本地运行

```bash
# 单平台采集
python run_crawler.py <平台标识> <关键词>

# 示例：采集抖音关于"闪充"的内容
python run_crawler.py dy 闪充
```

## GitHub Actions 使用

### 手动触发

1. 进入 GitHub 仓库的 Actions 页面
2. 选择"多平台数据采集"工作流程
3. 点击"Run workflow"按钮
4. 填写参数：
   - **平台选择**：选择要采集的平台（单选）
   - **搜索关键词**：输入搜索关键词
5. 点击"Run workflow"开始执行

### 工作流程步骤

1. **环境准备**：检出代码、设置 Python/Node.js 环境、安装依赖
2. **代码质量检查**：使用 flake8 检查代码
3. **数据采集**：运行采集程序获取数据
4. **数据打包**：生成 `关键词-日期.zip` 文件
5. **创建发布**：上传到 GitHub Releases

## 注意事项

1. **Cookie 配置**：确保在对应的平台配置文件中设置了有效的 Cookie，配置文件位于 `config/` 目录下（如 `config/dy_config.py`、`config/bilibili_config.py`）
2. **登录状态维护**：如果采集失败，系统会自动退出对应平台账号，重新登录获取新的 Cookie 并更新配置文件（`config/` 目录下对应平台的配置文件）
3. **数据保存**：数据默认保存在 `data` 目录下
4. **异常处理**：支持失败重试机制，单个平台失败不影响其他平台


## 故障排除

### Cookie 过期

如果采集失败并提示 Cookie 过期：
1. 系统会自动尝试重新登录获取新 Cookie
2. 如果自动登录失败，请手动更新对应平台配置文件中的 Cookie

### 网络问题

- 检查网络连接
- 确认目标平台可访问
- 考虑使用代理服务

### 数据为空

- 确认关键词是否正确
- 检查平台是否有相关内容
- 检查 Cookie 是否有效
