# 多平台爬虫工作流程使用说明

## 概述

这个GitHub Actions工作流程用于自动化爬取多个平台的数据，并将结果打包发布到GitHub Releases。

## 支持的平台

- `xhs` - 小红书
- `dy` - 抖音
- `ks` - 快手
- `bili` - B站
- `wb` - 微博
- `tieba` - 贴吧
- `zhihu` - 知乎

## 使用方法

### 1. 手动触发工作流程

1. 进入GitHub仓库的Actions页面
2. 选择"多平台爬虫"工作流程
3. 点击"Run workflow"按钮
4. 填写以下参数：
   - **平台选择**：选择要爬取的平台（单选）
   - **搜索关键词**：输入搜索关键词（如：闪充）
5. 点击"Run workflow"开始执行

### 2. 工作流程参数说明

#### platforms（平台选择）
- **类型**：单选
- **选项**：
  - `xhs` - 小红书
  - `dy` - 抖音
  - `ks` - 快手
  - `bili` - B站
  - `wb` - 微博
  - `tieba` - 贴吧
  - `zhihu` - 知乎
- **默认值**：`dy`（抖音）
- **说明**：每次只能选择一个平台

#### keywords（搜索关键词）
- **类型**：文本输入
- **默认值**：`闪充`
- **说明**：输入要搜索的关键词

## 工作流程步骤

### 1. 环境准备
- 检出代码仓库
- 设置Python 3.11环境
- 安装Python依赖包
- 设置Node.js 20.11.0环境
- 安装Playwright浏览器驱动

### 2. 代码质量检查
- 使用flake8检查Python代码语法错误
- 检查代码复杂度和行长度

### 3. 数据爬取
- 根据选择的平台更新配置文件
- 运行爬虫程序获取数据
- 支持异常处理，失败后继续执行

### 4. 数据打包
- 将爬取的数据打包成zip文件
- 文件命名格式：`关键词-日期.zip`（如：`闪充-20260414.zip`）
- 包含整个data目录的所有数据

### 5. 创建发布
- 创建GitHub Release
- Release名称：`关键词-日期`
- 使用唯一的run_id作为tag

### 6. 上传附件
- 将打包的zip文件上传到Release
- 用户可以直接下载获取数据

### 示例2：爬取小红书数据
- **平台**：选择`xhs`

- **结果**：生成`快充-20260414.zip`，包含小红书相关数据

### 示例3：爬取B站数据
- **平台**：选择`bili`

## 注意事项

1. **Cookie配置**：确保在对应的平台配置文件中设置了有效的Cookie
2. **数据保存**：数据默认保存在`data`目录下
3. **异常处理**：如果某个平台爬取失败，会继续执行其他平台
4. **文件命名**：打包文件名格式为`关键词-日期.zip`，日期格式为`YYYYMMDD`
5. **时区设置**：使用上海时区生成日期

## 本地使用

如果需要在本地运行相同的爬虫功能，可以使用以下命令：

```bash
# 单平台爬取
python run_crawler.py dy 闪充

# 多平台爬取（需要修改代码支持）
python run_crawler.py dy,ks 闪充
```

## 故障排除

### 问题1：爬虫失败
- **原因**：Cookie过期或无效
- **解决**：更新对应平台配置文件中的Cookie

### 问题2：没有生成数据文件
- **原因**：平台反爬虫机制或网络问题
- **解决**：检查网络连接，调整爬取参数

### 问题3：打包失败
- **原因**：data目录不存在或为空
- **解决**：检查爬虫是否正常运行，确认数据文件已生成

## 技术栈

- **Python 3.11**：主要编程语言
- **Playwright**：浏览器自动化
- **Node.js 20.11.0**：依赖环境
- **GitHub Actions**：CI/CD平台
- **flake8**：代码质量检查
